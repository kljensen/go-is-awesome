How We Went from 30 Servers to 2: Go
Our sustained CPU usage on our servers was approximately 50-60%... At some threshold above 50%, our Rails servers would spike up to 100% CPU usage and become unresponsive.

After we rolled out our Go version, we reduced our server count to two and we really only had two for redundancy. They were barely utilized, it was almost as if nothing was running on them. Our CPU utilization was less than 5% and the entire process started up with only a few hundred KB's of memory (on startup) vs our Rails apps which were ~50MB (on startup).

Basic Language Structure

  • Looks like C, but
  • Semicolons optional

Packages

  • Every piece of code lives in a packge
  • Imports give you access to other packages


package main import ( "fmt" "math" ) func main() { fmt.Println("Happy", math.Pi, "Day") }

Packages only export Capitalized names

That's how you control what is private / public


package main import ( "fmt" "math" ) func main() { // Won't compile, "pi" is lowercase/private fmt.Println("Happy", math.pi, "Day") }

Functions

Inputs and outputs typed, passed by value


package main import "fmt" func add(x int, y int) int { return x + y } func main() { fmt.Println(add(42, 13)) }

Functions can return multiple values


package main import "fmt" func sumdiff(x int, y int) (int, int) { var xysum, xydiff int xysum = x + y xydiff = x - y return xysum, xydiff } func main() { a, b := sumdiff(42, 10) fmt.Println(a) fmt.Println(b) }

Basic types

& variable initialization

Variables are statically typed

The type can be set explicity or implicitly


package main import "fmt" func main() { // Explicit var i int i = 5 // Implicit j := 4 fmt.Println(i + j) }

Basic built-in types


package main import "fmt" func main() { var i int i = 42 fmt.Println(i) var j float32 j = 3.14 fmt.Println(j) var isAwesome bool isAwesome = true fmt.Println(isAwesome) var x byte x = 'a' fmt.Println(x) }

Same code, using implicit format


package main import "fmt" func main() { i := 42 fmt.Println(i) j := 3.14 fmt.Println(j) isAwesome := true fmt.Println(isAwesome) x := 'a' fmt.Println(x) }

Arrays and slices

The former, fixed size, the latter, flexible


package main import "fmt" func main() { // Array, fixed size dayArray := [7]int {1, 2, 3, 4, 5, 6, 7} fmt.Println(dayArray) // Slice, flexible size daySlice := make([]int, 7) for i, day := range dayArray{ daySlice[i] = day } daySlice = append(daySlice, 121, 122, 123, 124) fmt.Println(daySlice) }

Maps

The keys and values can be any type


package main import "fmt" func main() { var timeZone = map[string] int { "UTC": 0, "EST": -5, "CST": -6, "MST": -7, "PST": -8, } fmt.Println(timeZone["EST"]) }

Structs

They're just collections of fields, as in C


package main import "fmt" type Rect struct { H int W int } func main() { r := Rect{1, 2} fmt.Println(r.H + r.W) }

Struct methods

The struct type precedes the function definition


package main import "fmt" type Rect struct { H, W int } func (r *Rect) Area() int { return r.W * r.H } func main() { rect := Rect{3, 6} fmt.Println(rect.Area()) }

All Go types have default values


package main import "fmt" type Rect struct { H, W int } func main() { var a int var b float var c [5]int var d = make([]int, 5) var e make(map[string]int) var f Rect fmt.Println(a, b, c, d, e, f) }
